# n8n WhatsApp Integration Workflow

This document describes the n8n workflow configuration for connecting WhatsApp to the AG Associates AI backend.

## Prerequisites

1. **Meta Developer Account**: Create at https://developers.facebook.com/
2. **WhatsApp Business API**: Set up a WhatsApp Business account in the Meta Developer Dashboard
3. **WhatsApp Sandbox**: For testing, use the sandbox environment

## Docker Setup

n8n is already configured in `docker-compose.yml`:

```yaml
services:
  n8n:
    image: n8n/n8n:latest
    container_name: ag-associates-n8n
    ports:
      - "5678:5678"
    environment:
      - N8N_HOST=localhost
      - N8N_PORT=5678
      - N8N_PROTOCOL=http
      - WEBHOOK_URL=http://localhost:5678/
    volumes:
      - n8n_data:/home/node/.n8n
```

Start n8n:
```bash
docker-compose up -d n8n
```

Access n8n at: http://localhost:5678

## Workflow Configuration

### Step 1: Create New Workflow

1. Open n8n at http://localhost:5678
2. Click "Create new workflow"
3. Name it: "AG Associates - WhatsApp Intake"

### Step 2: Add Webhook Trigger

**Node 1: Webhook (WhatsApp Incoming)**
- Node Type: `Webhook`
- Method: `POST`
- Path: `whatsapp-webhook`
- Response Mode: `Last Node`

Configuration:
```json
{
  "httpMethod": "POST",
  "path": "whatsapp-webhook",
  "responseMode": "lastNode",
  "options": {}
}
```

### Step 3: Parse WhatsApp Message

**Node 2: Function Item**
- Node Type: `Function Item`
- Name: `Parse WhatsApp Message`

Code:
```javascript
// Extract message details from WhatsApp webhook payload
const message = $input.first().json;

return {
  json: {
    message: message.messages?.[0]?.text?.body || message.entry?.[0]?.changes?.[0]?.value?.messages?.[0]?.text?.body,
    sender: message.messages?.[0]?.from || message.entry?.[0]?.changes?.[0]?.value?.messages?.[0]?.from,
    timestamp: new Date().toISOString(),
    metadata: {
      platform: 'whatsapp',
      raw_payload: message
    }
  }
};
```

### Step 4: Send to FastAPI Backend

**Node 3: HTTP Request**
- Node Type: `HTTP Request`
- Name: `Send to AG Associates AI`
- Method: `POST`
- URL: `http://host.docker.internal:8001/webhook/whatsapp`

Body (JSON):
```json
{
  "message": "{{ $json.message }}",
  "sender": "{{ $json.sender }}",
  "timestamp": "{{ $json.timestamp }}",
  "metadata": {{ JSON.stringify($json.metadata) }}
}
```

Headers:
```
Content-Type: application/json
```

### Step 5: Wait for Response

**Node 4: Wait** (Optional)
- Node Type: `Wait`
- Time: `2 seconds`
- Purpose: Allow backend to process and generate PDF

### Step 6: Send PDF Back to WhatsApp

**Node 5: HTTP Request (WhatsApp Media)**
- Node Type: `HTTP Request`
- Name: `Send PDF to WhatsApp`
- Method: `POST`
- URL: `https://graph.facebook.com/v18.0/{{ $json.phone_number_id }}/messages`

Body (JSON):
```json
{
  "messaging_product": "whatsapp",
  "to": "{{ $item(0).json.sender }}",
  "type": "document",
  "document": {
    "link": "{{ $json.document_url }}",
    "filename": "rental_agreement.pdf",
    "caption": "Your rental agreement is ready! Generated by AG Associates AI."
  }
}
```

Headers:
```
Authorization: Bearer YOUR_WHATSAPP_ACCESS_TOKEN
Content-Type: application/json
```

### Step 7: Error Handling

**Node 6: Error Trigger**
- Node Type: `Error Trigger`
- Purpose: Catch and log any failures
- Action: Send notification to admin

## Environment Variables

Set these in n8n or your `.env` file:

```bash
# WhatsApp API Credentials
WHATSAPP_ACCESS_TOKEN=your_access_token_here
WHATSAPP_PHONE_NUMBER_ID=your_phone_number_id
WHATSAPP_BUSINESS_ACCOUNT_ID=your_business_account_id

# Backend URL
BACKEND_URL=http://host.docker.internal:8001
```

## Testing the Workflow

1. **Start all services:**
   ```bash
   docker-compose up -d
   ```

2. **Activate workflow in n8n:**
   - Open n8n dashboard
   - Toggle workflow to "Active"

3. **Test with WhatsApp sandbox:**
   - Send a test message to your sandbox number
   - Check n8n execution log
   - Verify PDF generation in `/workspace/ag-associates-ai/output/`

4. **Monitor backend logs:**
   ```bash
   docker logs ag-associates-backend
   ```

## Production Deployment

For production:

1. Use HTTPS for all webhooks
2. Store credentials in n8n credentials manager
3. Set up proper error handling and notifications
4. Configure rate limiting
5. Enable workflow versioning
6. Set up backup for n8n data volume

## Troubleshooting

### Issue: Webhook not receiving messages
- Verify WhatsApp webhook URL is correctly set in Meta Developer Dashboard
- Check n8n is accessible from external network
- Ensure port 5678 is open

### Issue: Backend connection failed
- Use `host.docker.internal` instead of `localhost` in Docker
- Verify backend is running on port 8001
- Check firewall rules

### Issue: PDF not sent back
- Verify document URL is publicly accessible
- Check WhatsApp media upload limits (max 100MB)
- Ensure access token has media permissions
